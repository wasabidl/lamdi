# Lamdi – Frontend (Expo / React Native)

An Expo + React Native app targeting Android, iOS, and Web.

---

## Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| Node.js | 18+ | `node --version` |
| Java (JDK) | 17 | Required by the React Native Gradle plugin |
| Android SDK | API 35 | Only needed for native Android builds |

---

## Quick Start (Expo Go / Web)

No Android SDK needed — runs in the browser or via the **Expo Go** mobile app.

```bash
npm install
npx expo start          # opens Metro bundler
# press 'w' for web, or scan the QR code with Expo Go on your phone
```

---

## Local Android Build (debug APK)

### 1. One-time environment setup

Run the included setup script to install the Android SDK and required components:

```bash
chmod +x setup-android.sh
./setup-android.sh
```

The script installs:
- OpenJDK 17 (via apt / Homebrew)
- Android command-line tools
- Android SDK platform 35, build-tools 35.0.0, NDK 27.1.12297006

After it finishes, reload your shell (or `source ~/.bashrc`) so `ANDROID_HOME` is on your path.

### 2. Build the debug APK

```bash
npm run build:android
```

This runs `expo prebuild` (regenerates the `android/` folder) then compiles via Gradle.  
The APK lands at:

```
android/app/build/outputs/apk/debug/app-debug.apk
```

### 3. Install on a device / emulator

```bash
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

Or drag-and-drop the APK onto a running Android emulator.

## Running on a connected Android device (live reload)

Make sure the device has **USB debugging** enabled and is visible to `adb devices`, then:

```bash
npm run android          # builds & deploys, opens Metro for live reload
```

## Available scripts

| Command | Description |
|---------|-------------|
| `npm start` | Start Metro bundler (Expo Go / web) |
| `npm run android` | Build & run on a connected device/emulator |
| `npm run ios` | Build & run on iOS simulator (macOS only) |
| `npm run web` | Start the web version |
| `npm run build:android` | Build a standalone debug APK |
| `npm run lint` | Run ESLint |

Copy `.env.example` to `.env.local` (if present) and fill in any required values before starting the app.

## Project structure

```
frontend/
├── app/          # File-based routes (Expo Router)
├── src/          # Shared components, hooks, utils
├── assets/       # Images, fonts
└── android/      # Generated native project (do not edit manually)
```

The `android/` folder is generated by `expo prebuild` and should not be edited by hand. Re-run `npm run prebuild:android` whenever you upgrade Expo or add a native module.
